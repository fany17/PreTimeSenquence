# 交易预测逻辑审查

## 旧逻辑的核心问题

1. `B_GotPosition.py` 用未来完整路径生成 `position_signal`，但没有最大持仓期。越往未来扫，越容易总有一边先触发，标签会变成事后归因，而不是可交易预测目标。
2. 旧 GT 将“如果做多先止损”直接标成 `short`，将“如果做空先止损”直接标成 `long`。这不是同一笔可执行交易的真实收益，而是用事后路径选择方向。
3. 同一根 K 线内同时触及上下边界时，旧代码按检查顺序决定标签，隐含了不可观测的盘中先后顺序。
4. `train_test_split` 随机切分时间序列，会把相邻行情 regime 同时放进训练和测试，导致验证结果虚高。
5. 旧标签没有显式扣除手续费、滑点和最低收益门槛，模型可能学习到无法覆盖交易成本的微小波动。
6. `long/short` 二分类没有 `flat`，会强迫模型每个时点都出手，实盘里这通常会放大噪声交易。

## 新逻辑

当前重构把任务定义为三分类：

- `long`: 信号形成后，未来有限 horizon 内上行 barrier 先被触发，或 horizon 收益扣成本后足够为正。
- `short`: 下行 barrier 先被触发，或 horizon 收益扣成本后足够为负。
- `flat`: 没有足够边际，不交易。

默认参数：

- horizon: 20 根 K 线
- barrier: `max(4 * ATR / close, 0.5%) + round_trip_cost`
- round-trip cost: 手续费和滑点双边扣除
- split: 训练、验证、测试按时间顺序切分

## 当前诊断结果

在 `data/market_data_DOGE_new.pkl` 上，默认标签分布约为：

- short: 23120
- long: 16645
- flat: 11516

第一次按时间顺序训练 XGBoost 三分类模型后，测试集 balanced accuracy 约为 `0.333`，接近随机三分类。这说明旧绝对价格/均线特征没有证明有稳定预测能力；旧结果如果明显更好，优先怀疑标签和验证方式导致的乐观偏差。

后续已将特征改为收益、波动率、ATR、均线相对偏离、布林 z-score、量能 z-score 和日内周期项，并加入类别加权训练。当前 DOGE 35.6 天样本上的单次时间测试 balanced accuracy 约为 `0.352`，walk-forward 分段约为 `0.350-0.365`。模型不再坍缩到单一方向，但仍只略高于随机三分类，不能证明有可实盘使用的稳定 alpha。

调用本地 Binance API key 拉取 `DOGEUSDT` 从 2024-01-01 开始的 300000 根 1m K 线后，样本扩展到约 208.3 天。时间测试 balanced accuracy 提升到约 `0.429`，walk-forward 四段约为 `0.375 / 0.396 / 0.430 / 0.430`，明显好于 majority baseline `0.333` 和 momentum20 baseline `0.347`。这说明此前“数据太少”是重要原因之一，但早期 fold 仍弱，regime shift 仍然明显。

## 下一步

1. 先不要接实盘下单，直到 walk-forward 回测能稳定覆盖手续费和滑点。
2. 先扩展数据跨度，至少覆盖半年到一年，当前 35.6 天单币种数据太短。
3. 对 `BTC/ETH/DOGE` 分别做同一套标签和 walk-forward 验证，不要在一个币种上调参后直接宣称有效。
4. 增加基线：buy-and-hold、永远 flat、动量、均值回归。模型必须超过这些基线才有意义。
5. 用概率阈值控制出手频率，例如只在 `max_prob > 0.55/0.60` 时交易。
